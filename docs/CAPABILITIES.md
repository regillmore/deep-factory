## World Streaming And Meshing

- Chunk visibility culling with padded chunk streaming retention.
- Chunk pruning can evict resident world chunks from memory without losing edited tile values; sparse per-chunk tile overrides are reapplied when those chunks stream back in.
- Resident world chunks now also own zeroed per-chunk `Uint8Array` light fields plus dirty-light flags and dirty-local-column masks, and newly streamed chunks start with all local sunlight columns dirty so resident lighting caches can be rebuilt from tile state; draw-range chunks are made resident before the renderer's relight pass so visible stream-back chunks do not build once with zeroed dirty light and then immediately rebuild.
- Tile edits now clear and dirty only the loaded local sunlight columns affected by that edit whenever the edited tile changes `blocksLight` or emissive-light state; emissive source edits widen that invalidation to nearby columns within the edited source's emissive radius, zero-range non-emissive `blocksLight` edits now also invalidate the edited column's immediate neighbors so one-tile-gap blocker-face relighting recomputes adjacent columns, non-emissive `blocksLight` edits still widen invalidation when nearby resident emissive sources can still reach the edit, and non-emissive edge edits additionally invalidate loaded neighboring `chunkX` boundary columns plus their immediately adjacent interior columns so horizontal sunlight transport and boundary-adjacent solid-face relighting recompute both sides of resident chunk edges.
- Dirty resident light fields are recomputed during rendering by propagating sunlight top-down from exposed resident chunk tops only within resident `chunkX` columns that currently contain dirty light chunks, then transporting sunlight across loaded neighboring `chunkX` boundary columns when dirty edge columns are recomputed from sunlight-only boundary visibility rather than retained emissive falloff in clean neighbors, then writing sunlight onto blocking tiles that are cardinally adjacent to sunlit non-blocking tiles in those dirty columns (including recessed one-tile vertical air pockets whose horizontal neighbors are sunlit), then layering local emissive-source cardinal falloff over that sunlight base for those same dirty local columns; both sunlight and emissive recomputation write onto the first blocking tile in each lit path so exposed or adjacent solid faces stay visible while gameplay `blocksLight` metadata still terminates further propagation behind those blockers before rebuilt local columns are marked clean.
- World chunk rendering now modulates tile atlas color by resolved per-tile light values baked into chunk-mesh vertices, and dirty-light chunk recomputation now invalidates only cached chunk meshes whose light arrays actually changed, including neighboring boundary chunks touched by horizontal sunlight transport, so lighting-only changes reupload without broad relight-driven queue churn.
- Resident water and lava now keep per-tile `0..8` fill levels in world state and step in the fixed `60hz` update by transferring downward only from currently awake resident active-liquid chunks first, then sideways through deterministic loaded-chunk pair equalization that only scans the resulting active-liquid chunk band plus immediate horizontal neighbors; settled liquid chunks now stay resident but sleep after two quiet ticks so the alternating sideways-pair parity is preserved before they drop out of the active set, and local edits or nearby liquid transfers wake neighboring resident liquid chunks again. `TileWorld` still rebuilds awake liquid membership from resident chunk contents on snapshot load or stream-back, so completely dry resident worlds and fully sleeping settled pools both skip the liquid step until a disturbance reintroduces awake work. ## Hot-Path Lookup Strategy

- Gameplay metadata compiles into dense lookup arrays for collision, light-blocking, emissive-light, and liquid queries.
- Terrain connectivity compiles into dense connectivity-group and material-tag lookup tables.
- Tile render metadata compiles into dense static UV and terrain variant atlas-index lookup tables.
- Liquid connectivity compiles into dense per-tile group ID and label lookups, and liquid variant metadata compiles into per-tile cardinal-mask render lookups consumed directly by the liquid meshing path.
- Optional animated render metadata compiles into dense per-tile frame start, frame count, and frame duration tables plus a flattened UV-frame list for renderer-time elapsed-frame sampling.
- Optional animated liquid variant metadata also compiles into dense per-tile-per-cardinal-mask frame start, frame count, and frame duration tables plus a flattened UV-frame list for renderer-time liquid elapsed-frame sampling.
- Placeholder terrain autotile resolution uses precomputed lookup tables for normalized adjacency masks and raw adjacency masks.
- Authored-atlas region UV rect objects are precomputed and reused by atlas-index and terrain-autotile resolution paths.
